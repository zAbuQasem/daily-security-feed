Comprehensive technical overview of GCP-specific offensive security attack vectors, addressing the skills gap in cloud penetration testing. Covers service account impersonation via IAM permissions, Cloud Build privilege escalation through default service account permissions, Workload Identity abuse and metadata server access in GKE, BigQuery data exfiltration techniques, Cloud Functions/Cloud Run authentication bypass, and cross-product attacks leveraging Google Workspace domain-wide delegation. The article emphasizes that GCP's resource hierarchy, IAM binding inheritance, and service account architecture differ fundamentally from AWS/Azure, making ported attack patterns ineffective—practitioners must understand GCP-specific weaknesses to conduct complete assessments.
