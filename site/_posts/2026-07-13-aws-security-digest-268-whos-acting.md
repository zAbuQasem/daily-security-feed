AWS Security Digest #268 curates three substantive AWS security deep-dives: Aidan Steele reverse-engineers IAM Identity Center's new programmatic access feature, documenting CloudTrail event signatures and readiness considerations for delegated user role assumption. Nathanael Ndong provides an AWS forensics primer covering the account and identity model, CloudTrail retention windows (90 days by default), evidence data events that require explicit enablement, and techniques for reconstructing attacker activity from recon through IAM backdoors to Lambda persistence. Sena Yakut analyzes GuardDuty as a first-line detection service spanning CloudTrail, VPC Flow Logs, DNS logs, and broader sensor coverage, while acknowledging remaining detection gaps. The digest also references Microsoft's recent MCP tool description poisoning research showing AI agents can exfiltrate data while technically following all policies.
